# journal/

日々の作業の生記録。non-stop 作業や長時間セッションの後、膨大なログを追うより journal を読み返す方が状況復元しやすい。

## 命名

`YYYY-MM-DD-topic.md`

複数 topic を 1 日にまたがって書く場合は別ファイルに分ける。

## 中身の粒度

- ハマり所 → 解決策のペア
- 設定値・コマンド・変更点（後日コピペで再現できる程度）
- 議論の要点（結論だけでなく、なぜその結論になったかの経緯）

`docs/findings/` は「確定事実」中心、こちらは「経緯と試行錯誤」中心。

- `~/.claude/rules/docs-structure.md` の `journal/` 項
